# Agri Port — アプリDE農業(ハッカソンDEMO)

フィジカルAIが耕し、人間はアプリで観察しながら投資的に参加する——
農業を「誰でも参加できるアクティビティ」に変えるビジネスのデモサイト。

> **🌐 Live Demo:** (Cloudflare Pages デプロイ後にURLをここに記載 — 例: `https://agri-port.pages.dev`)
> 設定: CFダッシュボード → Workers & Pages → リポジトリ接続 → ビルドコマンド空欄 / 出力ディレクトリ `/`

## 構成(ビルド不要・静的サイト)

| ファイル | 内容 |
|---|---|
| `index.html` | ランディング: 統計データによる課題提起 → 本質課題の再定義 → ソリューション → ビジネスモデル(2トラック規制設計) → Why Now → ロードマップ2035 |
| `app.html` | アプリ本体デモ: ポートフォリオ / ライブ観察(カメラ3視点: 定点・ロボットPOV(AI雑草検出HUD)・ドローンNDVI) / サプライチェーン・トラッキング(ヒューマノイド選果) / リターン選択(野菜⇄お金) / 募集中ポート |
| `assets/` | 共通CSS・チャートライブラリ(自作SVG)・デモロジック |
| `assets/images/` | AI生成ビジュアル(PNG原本)。サイトが参照するのは `optimized/` のWebP(1600px/900px幅・各100〜330KB)。使い分けは `IMAGE_USAGE_NOTES.md` 参照 |

## 起動方法

```bash
open index.html          # そのままブラウザで開くだけ
# または
python3 -m http.server   # → http://localhost:8000
```

## プレゼンの流れ(推奨)

1. **index.html** — ヒーロー → 「担い手減少は人手不足ではなく設計問題」の統計パート(チャートはホバーで値が出ます)
2. スキーム図と **Track A(野菜=金商法外)/ Track B(金銭=二種業提携)** の段階設計
3. **app.html** へ — ライブ観察でカメラを「ロボット視点」に切り替え、AI雑草検出ボックスがリアルタイムに出る様子を見せる(→ドローンNDVIで生育ムラ検出も)。リターンセンターのトラッキングで「収穫→ヒューマノイド選果→直送」の48時間サプライチェーンを見せる
4. リターン切替(野菜⇄お金)→ 募集中ポートで「出資する」を押してデモ完結

## データの出典

統計はすべて一次ソース(農林水産省 農林業センサス2025概数値・農業構造動態調査・新規就農者調査・荒廃農地調査、NEDO、矢野経済研究所など)。各チャート直下に出典を明記。

> 本サイトはデモです。記載の分配率・募集はすべて架空であり、金融商品の勧誘ではありません。
